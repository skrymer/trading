# Trading Project Memory

## User Profile

- [Single-user app](user_single_user_app.md) — the developer is the sole operator; manual triggers are fine, no need for dashboards / cascades / admin UIs

## User Preferences

- **Only strategies are named, not conditions**: [feedback_only_strategies_are_named.md](feedback_only_strategies_are_named.md) — Norse-god names (VCP, Fenrir, …) are for assembled strategies; a condition screened in isolation stays referred to by its type. Name the strategy only when it's built around the condition.
- **Always fix pre-existing issues**: Broader than lint — includes security findings (API key leaks), perf bugs, race conditions, and any other Critical/High issues surfaced during code review or pre-commit. Don't defer them as "follow-ups" or "out of scope" just because they're not in the diff. Fix them in the current commit, or split into a tightly-scoped follow-up commit before merging. Reason: leaving Critical/High findings uncommitted creates a backlog of known issues that drift across sessions and never get addressed.
