# Fundamentals are point-in-time reference data gated on `filing_date`; quality is a cross-sectional percentile mirroring ADR 0009

To build a cross-sectional quality/profitability long premise — the one factor whose edge plausibly *survives* narrow-leadership tape rather than participating-and-losing or delivering beta (`knowledge/purpose.md`) — we add two layers, each mirroring an existing precedent rather than inventing a new shape. **L1:** Midgaard ingests EODHD quarterly Income-Statement + Balance-Sheet line items as **point-in-time reference data** — one record per `(symbol, fiscalDateEnding)` stamped with its **`filing_date`** — shipped to Udgaard as `Stock.fundamentals` exactly like **earnings**, with visibility gated on `filing_date ≤ tradingDate` (never `fiscalDateEnding`). **L2:** Midgaard computes a **market-wide, survivorship-free quality percentile** in a separate operator-triggered cross-sectional pass, persisted per quote row and read by a fail-closed condition — exactly mirroring **ADR 0009** (the market-relative-strength percentile). See CONTEXT.md *Point-in-time fundamentals* for the term.

## Why

- **Why fundamentals, why now.** Every prior long premise died as [participate-and-lose] or [beta-delivery] because it conditioned on *price* state, which in a strong tape is just market direction. A quality measure is computed from financial statements — non-price, persistent, and structurally favoured when leadership concentrates into high-profitability mega-caps. The full data picture and the dead-class reasoning live in `knowledge/`.
- **Why `filing_date`, not `fiscalDateEnding`.** A fiscal period ends 1–3 months before the report is public; ranking on `fiscalDateEnding` would leak that gap into *every* observation. Gating on `filing_date` is the first-order look-ahead guard, identical in spirit to the earnings `reported_date` guard (`Earning.isWithinDaysOf`).
- **Why Midgaard, and a separate pass.** A market-wide rank needs the complete survivorship-free universe, which only Midgaard holds (ADR 0011 — Udgaard has no symbol catalogue). The cross-sectional stage cannot live in the per-symbol ingest loop (a symbol's percentile on date D depends on every other symbol's latest-known fundamental on D), exactly as ADR 0009 established for relative strength.

## Considered options

- **Ship raw fundamentals to Udgaard AND persist a market-wide percentile (chosen).** The two serve disjoint roles, reusing ADR 0009's gate-vs-ranker distinction: the **Midgaard percentile is an absolute, market-wide gate** (per-symbol, independent of the day's pool → a trivial fail-closed eligibility condition); the **Udgaard ranker is a relative, intra-subset ordering** computed from raw `Stock.fundamentals` (no full universe needed, like `TrailingReturnRanker`). The ranker can iterate freely with no re-pass; only the gate metric is baked into Midgaard.
- **Single baked gate metric, re-run the pass to iterate it (chosen)** — over persisting several metric percentiles. The re-run reads already-stored raw fundamentals (no EODHD re-fetch) and is already operator-gated; baking one metric keeps the schema clean and matches how RS was done (ADR 0009 fixed a single 252-bar metric and deferred variants). Ordering-dimension iteration speed is preserved anyway via the Udgaard ranker.
- **Point-in-time vintages — declined.** EODHD serves *latest-restated* values keyed to each fiscal period with the original `filing_date`, not as-first-reported vintages. The `filing_date` gate is accepted as the guard and the **restatement residual is documented** (a bounded second-order leak, far smaller than the first-order gap the gate kills, on a slow/broad measure). Eliminating it would require a PIT-vintage vendor — revisited only if the premise survives and the residual proves material.
- **Compute the percentile in Udgaard — rejected.** Udgaard has no survivorship-free universe (ADR 0011) and it would duplicate the cross-sectional pipeline outside the service that owns the universe (the same rejection ADR 0009 recorded). `BacktestContext` per-date precompute is likewise rejected (wrong peer set + full-universe OOM, per ADR 0009).

## Consequences

- The L2 quality-percentile pass is **operator-triggered**, not auto-fired post-ingest, for the same cost reason as ADR 0009 (the whole-universe sort is multi-minute). A re-ingest leaves the field **null until recompute**; the condition is **fail-closed on null** (admits no entry on un-ranked bars rather than ranking against a stale or incomplete cross-section). Recommended as a *separate* recompute endpoint, parallel to the RS recompute, so the quality metric can be re-run without touching RS.
- The pass adds one mechanic over RS: an **as-of join** — for each date D, each symbol contributes its latest fundamental with `filing_date ≤ D` (the quarterly value forward-filled across daily bars, gated on `filing_date`), then ranked cross-sectionally. Definedness, the midpoint estimator, the `N_min` peer-count floor, and the hard 2000-01-01 calendar floor are inherited verbatim from ADR 0009.
- **Inherits ADR 0009's two verification gates before any firewall run:** `min(N)` over 2000–2002 must clear `N_min` on every trading day (else those dates fail-closed for all symbols — acceptable but must be *known*), and PIT/bit-exactness verified on PRD. For fundamentals the binding coverage question is whether enough smaller-cap names carry statements back to 2000 to clear `N_min`.
- The `EodhdFundamentalsClient` `@Cacheable` key is `symbol`, so existing cached responses won't contain the new `Financials` sections — the re-ingest must bust that cache.
- **Out of scope of this ADR (quant-owned, decided before code):** the exact gate metric (gross profitability GP/TA vs ROE vs a composite), the Udgaard ranker composite + margin-trend leg, percentile/threshold values, and therefore the strategy-neutral condition/ranker names (named after the locked mechanic). This ADR fixes the data model and the architectural shape, not the signal definition.
